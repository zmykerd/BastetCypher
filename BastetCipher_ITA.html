⚠ Best practice — Sicurezza Operativa

      

        

          
🔴 Disabilita l'ibernazione

          

            Quando il PC va in ibernazione, l'intera RAM viene scritta su disco.

            Eventuali dati aperti nel Caveau finirebbero nel file di ibernazione.


            Windows: powercfg /h off

            macOS: sudo pmset -a hibernatemode 0

            Linux: rimuovi la riga resume= da GRUB e disabilita la partizione di swap dall'ibernazione.
          

        

        

          
🔴 Disabilita la Swap / Memoria virtuale

          

            Se la RAM è piena, il sistema può spostare pagine di memoria su disco (swap/pagefile).

            Questo può includere dati decrittografati anche senza ibernazione.


            Windows: Sistema → Impostazioni di sistema avanzate → Prestazioni → Memoria virtuale → Nessun file di paging.

            macOS: La swap è gestita automaticamente; usa FileVault per cifrare il disco, così la swap è cifrata.

            Linux: sudo swapoff -a (temporaneo) oppure rimuovi le righe swap da /etc/fstab.
          

        

        

          
🟡 Cifra il disco (se swap non disattivabile)

          

            Se non puoi disattivare la swap, assicurati che l'intero disco sia cifrato.

            In questo modo anche i dati finiti nella swap restano inaccessibili senza la chiave di avvio.


            Windows: VeraCrypt (BitLocker è sconsigliato)

            macOS: FileVault 2

            Linux: LUKS con dm-crypt
          

		  
		   

            ℹ
            

              Perché non BitLocker?
              È closed source e non verificabile indipendentemente.
              Per impostazione predefinita, Windows carica la chiave di ripristino
              sull'account Microsoft — su server di terze parti fuori dal tuo controllo.
              Microsoft è soggetta alla legislazione statunitense (FISA, NSL) che può imporre
              la consegna di dati e chiavi con obbligo di segretezza verso l'utente finale.
              VeraCrypt è open source, sottoposto ad audit indipendenti pubblici,
              senza account, senza cloud e senza alcuna chiave su server esterni.
              La scelta è tua — ma ora sai.
            

          

		  
        

        

          
🟡 Cancella i file originali in modo sicuro

          

            Dopo aver creato l'archivio .bca, elimina i file originali non crittografati con una cancellazione sicura a più passate, per renderli irrecuperabili anche con strumenti forensi.


            Windows: Eraser (algoritmo Gutmann 35 passate o DoD 5220.22-M)

            macOS: rm -P file oppure Permanent Eraser

            Linux: shred -vuz -n 35 file (Gutmann) oppure wipe, altrimenti con peazip


            ⚠ Su SSD/NVMe la cancellazione sicura per sovrascrittura è meno affidabile per via del wear leveling — usa la cifratura del disco come protezione primaria (altrimenti bisogna fare la pulizia spazio libero e prendere ogni volta l'intero disco col rischio di usurarlo prematuramente).

          𓆣 Architettura Crittografica — Dettaglio Tecnico
        

        

          

            
Passo I — Sale Deterministico

            
SHA-256 Multiparametrico

            
Il sale è generato da SHA-256(frase + PIM + PEPPER + "SacredSalt"), rendendo ogni sessione crittograficamente unica senza dipendere da casualità esterna.

          

          

            
Passo II — Hash Stratificati

            
SHA-256 + SHA-384 + SHA-512

            
Tre hash in parallelo con ordini di concatenazione invertiti producono 192 byte di entropia grezza prima ancora della derivazione chiave.

          

          

            
Passo III — Trasformazione Proprietaria

            
4 Stadi · LCG + Fisher-Yates

            
Rotazione sinistra, swap posizionale, rimappatura nibble hex con permutazione LCG-seeded, inversione sezioni deterministiche. Rompe qualsiasi pattern strutturale.

          

          

            
Passo IV — Derivazione Chiave

            
PBKDF2-HMAC-SHA-512

            
Da 50.000 a 600.000+ iterazioni variabili in base a SHA-256(PIM), con twist XOR unico per ogni valore PIM. Rende impraticabile qualsiasi GPU-brute-force.

          

          

            
Passo V — Inserimento Glifi

            
PRNG Seeded + Mixed Case 50/50

            
8–15 caratteri speciali in posizioni deterministiche + capitalizzazione Fisher-Yates con seed invertito garantisce massima entropia visiva nel risultato finale.

          

          

            
Amplificatore Entropico

            
SHA-512 · LCG · 0–9999 caratteri

            
Estende deterministicamente il cifrario di N caratteri da SHA-512(frase§PIM§N§derivedKey§PEPPER). Stesso set di parametri → stessa estensione. Sempre.

          𓃠 Come funziona — Per tutti
        

        

          
Immagina di voler trasformare una frase che solo tu conosci — come "Il mio gatto dorme sul tetto" — in una password impossibile da indovinare, lunghissima, piena di simboli, lettere maiuscole e minuscole, numeri. Non una password qualunque: sempre la stessa identica password, ogni volta che inserisci la stessa frase. Come se l'universo ricordasse la tua scelta per sempre.

          
È esattamente quello che fa BastetCipher. La tua frase segreta viene macinata attraverso una catena di algoritmi crittografici militari — gli stessi usati da banche e governi — mescolata con un tuo numero personale chiamato PIM, trasformata, contorta, ribaltata e ri-derivata centinaia di migliaia di volte. Il risultato è un cifrario così lungo e complesso che nessun computer al mondo, neanche il più potente, potrebbe mai indovinarlo in una vita umana.

          
La cosa più geniale? Non salva nulla da nessuna parte. Niente database, niente cloud, niente trasmissioni internet. Tutto avviene solo nel tuo browser, sul tuo dispositivo, in totale silenzio. Se anche qualcuno rubasse questo file, non troverebbe le tue password. Le troverà solo tu, con la tua frase e il tuo PIM.

          
L'Amplificatore Entropico aggiunge un ulteriore strato: inserendo un numero da 0 a 9999, il cifrario si allunga esattamente di quei caratteri — generati in modo deterministico e crittograficamente legato a tutti i tuoi input. Più lungo è il cifrario, più infinita è la sua sicurezza.
